## 6. Выявленные проблемы

### 🔴 Критические

1. **Очень медленная обработка**
   - Среднее время: 49 секунд
   - Некоторые документы занимают до 10 минут
   - **Причина**: Большой размер документов, отсутствие оптимизации LLM запросов

2. **Документы с ошибками обрабатываются долго**
   - 6 из 10 самых медленных — с ошибками
   - **Причина**: LLM тратит время на генерацию невалидного ответа, который затем отбрасывается

3. **Низкий процент извлечения ИНН**
   - Только 39.4% от найденных победителей имеют ИНН
   - **Причина**: Либо ИНН отсутствует в документе, либо LLM его не извлекает

### 🟡 Важные

4. **Новые документы не обрабатываются**
   - Загружено 773 новых документа
   - Не обработано ни одного из них
   - **Причина**: Скрипт тестирования использует существующие результаты в qa_results

## 7. Рекомендации по оптимизации

### Производительность

1. **Добавить timeout для LLM запросов**
   ```python
   timeout=120  # 2 минуты максимум
   ```

2. **Раннее обнаружение проблемных документов**
   - Проверять размер документа до обработки
   - Если > N символов — разбивать на части или пропускать

3. **Оптимизация промптов**
   - Укоротить системный промпт
   - Использовать более точечные запросы

4. **Параллельная обработка**
   - Увеличить concurrency для batch обработки
   - Текущий: 5-10 параллельных запросов

### Качество

5. **Улучшить извлечение ИНН**
   - Добавить отдельный промпт для извлечения ИНН
   - Использовать регулярные выражения для пост-обработки

6. **Обработка ошибок**
   - Retry с другим промптом при JSON parse errors
   - Fallback на более простую модель
