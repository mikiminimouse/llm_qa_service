## 4. Выявленные проблемы

### 4.1 Критические

| ID | Проблема | Влияние |
|----|----------|---------|
| P1 | Ненадёжный парсинг JSON | 10.7% ошибок |
| P2 | LLM не возвращает JSON | 5.1% документов |
| P3 | Трейлинг-запятые в JSON | 4.9% ошибок |

### 4.2 Описание проблем

**P1: Ненадёжный JSON extraction**
- Регулярное выражение `\{.*\}` greedily захватывает текст
- Нет валидации JSON до парсинга
- Нет fallback логики

**P2: LLM не возвращает JSON**
- LLM иногда возвращает текст вместо JSON
- Нет обработки markdown блоков без `json` тега

**P3: Синтаксические ошибки в JSON**
- LLM генерирует невалидный JSON
- Распространённые проблемы: трейлинг-запятые, одинарные кавычки

## 5. Внедрённые улучшения

### 5.1 Улучшение JSON Parser

**Файл:** `application/response_parser.py`

**Изменения:**
1. Добавлена проверка валидности JSON перед возвратом
2. Улучшено регулярное выражение для balanced braces
3. Добавлен fallback метод `_extract_json_fallback()`
4. Добавлен метод `_fix_json()` для исправления распространённых проблем

```python
# До
JSON_OBJECT_PATTERN = re.compile(r"\{.*\}", re.DOTALL)

# После
JSON_OBJECT_PATTERN = re.compile(r"\{(?:[^{}]|(?:\{[^{}]*\}))*\}", re.DOTALL)
```

### 5.2 Фиксируемые проблемы

| Проблема | Фикс |
|----------|------|
| Trailing commas | `re.sub(r",(\s*[}\]])", r"\1", fixed)` |
| Single quotes | `fixed.replace("'", '"')` |
| Unquoted keys | `re.sub(r"(\{)\s*([a-zA-Z_][a-zA-Z0-9_]*)\s*:", r'\1"\2":', fixed)` |
