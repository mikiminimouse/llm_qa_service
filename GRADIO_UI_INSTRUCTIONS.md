# Инструкции для работы с Gradio Web UI

## Доступ с локальной машины (SSH проброс портов)

### С локальной Ubuntu машины

```bash
# Проброс порта 7860 (Gradio WebUI)
ssh -L 7860:localhost:7860 user@remote-server

# После подключения в браузере открыть:
# http://localhost:7860 — откроет WebUI удалённого сервера
```

### Проброс нескольких портов (WebUI + API)

Если нужно пробросить и API (8001):

```bash
# Проброс двух портов одновременно
ssh -L 7860:localhost:7860 -L 8001:localhost:8001 user@remote-server
```

### Фоновый проброс (без занятия терминала)

```bash
# Создаёт проброс в фоновом режиме
ssh -fN -L 7860:localhost:7860 user@remote-server

# Остановить фоновый проброс:
pkill -f "7860:localhost:7860"
```

### Сохранение проброса (SSH config)

Добавить в `~/.ssh/config` на локальной машине:

```
Host llm-server
    HostName your-server-ip
    User your-username
    LocalForward 7860 localhost:7860
    LocalForward 8001 localhost:8001
```

Использование: `ssh llm-server`

## Структура интерфейса

### 1. Главная таблица результатов

Отображает все обработанные документы с колонками:
- **unit_id** — уникальный идентификатор документа
- **winner_found** — найден ли победитель (✅/❌)
- **winner_name** — название организации-победителя
- **winner_inn** — ИНН победителя (10 или 12 цифр)

### 2. Фильтры

| Фильтр | Описание |
|--------|----------|
| **Все** | Показать все документы |
| **Победитель найден** | Только с найденным победителем |
| **С ИНН** | Только документы, где ИНН извлечён |
| **Без ИНН** | Документы, где ИНН не найден |
| **Победитель не найден** | Документы без победителя |

### 3. Поиск

Введите название организации или ИНН для поиска по результатам.

### 4. Детали документа

Кликните на строку в таблице, чтобы увидеть:
- Полное название победителя
- ИНН, КПП, ОГРН (если есть)
- Цену контракта
- Reasoning (обоснование LLM)
- Source quote (фрагмент текста)

### 5. Экспорт

Нажмите кнопку **"Экспорт в CSV"** для выгрузки всех результатов.

## Проверка качества работы

### Шаг 1: Откройте UI
```
http://localhost:7860
```

### Шаг 2: Проверьте статистику
- В таблице должны быть видны сотни записей
- Используйте фильтр "С ИНН" — должны быть записи (~287)
- Используйте фильтр "Без ИНН" — должны быть записи

### Шаг 3: Проверьте конкретные документы

1. Выберите фильтр "С ИНН"
2. Кликните на любую строку
3. Проверьте детальную карточку:
   - ИНН должен быть 10 или 12 цифр
   - Название организации должно быть полным
   - Reasoning должен объяснять, как найден победитель

### Шаг 4: Экспортируйте и проверьте

1. Нажмите "Экспорт в CSV"
2. Откройте файл в Excel
3. Проверьте:
   - Количество строк с ИНН
   - Формат ИНН (10 или 12 цифр)
   - Соответствие названия и ИНН

## API для статистики

### Получить статистику через API

```bash
curl -s "http://localhost:8001/api/v1/qa/stats" | jq
```

**Пример ответа:**
```json
{
  "total": 447,
  "winner_found": 287,
  "winner_not_found": 160,
  "service_files": 66,
  "with_errors": 0
}
```

### Получить список всех документов

```bash
curl -s "http://localhost:8001/api/v1/qa/documents" | jq
```

### Обработать один документ

```bash
curl -X POST "http://localhost:8001/api/v1/qa/process" \
  -H "Content-Type: application/json" \
  -d '{"unit_id": "UNIT_XXX", "force": false}'
```

### Пакетная обработка

```bash
curl -X POST "http://localhost:8001/api/v1/qa/process/batch-parallel-retry" \
  -H "Content-Type: application/json" \
  -d '{
    "unit_ids": ["UNIT_XXX", "UNIT_YYY"],
    "max_concurrent": 3,
    "retry_failed": true,
    "retry_delay_seconds": 30
  }'
```

## Типовые случаи

### Типы победителей, которые система извлекает:

| Тип | Пример |
|-----|--------|
| ООО | ООО «ПР-Сибирь», ИНН: 2465171595 |
| ИП | ИП Дамиров Артём Тимурович, ИНН: 860332447272 |
| АО | АО «КАМЕНСК-УРАЛЬСКОЕ АВТОТРАНСПОРТ...», ИНН: 6665000227 |

### Причины отсутствия победителя:

- Анонимизированные данные (нет ИНН в документе)
- Документы не являются протоколами вскрытия/оценки
- Многостраничные PDF с низкой OCR точностью
