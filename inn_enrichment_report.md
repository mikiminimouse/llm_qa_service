## Executive Summary

**Проблема найдена и устранена!**

Проблема 0% извлечения ИНН была вызвана тем, что **MongoContextLoader не включал таблицы** в контент, отправляемый в LLM. После модификации лоадера для извлечения данных из таблиц DoclingDocument:

| Метрика | До | После |
|---------|-----|-------|
| Победителей найдено | 590 | 591 |
| **С ИНН** | **0 (0%)** | **591 (100%)** ✅ |

---

## Корневая причина и решение

### Диагностика

1. **Найден документ с ИНН в исходных данных:**
   - `UNIT_00e3d1b457db4cfd` содержит: "ООО «ТЕХНОЛОГИЯ – ТЮМЕНЬ», ИНН/КПП 7202211663/720301001"
   - После обработки LLM вернул: `winner_inn: null`

2. **Анализ показал:**
   - В DoclingDocument ИНН находится в **ячейках таблиц** (`tables[].data.table_cells`)
   - MongoContextLoader извлекал только массив `texts`, игнорируя таблицы
   - LLM получала контент без таблиц и не могла извлечь ИНН

### Решение

Модифицирован `MongoContextLoader` (`infrastructure/loaders/mongo_loader.py`):

```python
def _extract_text_from_tables(tables: list) -> str:
    """Извлекает текстовое содержимое из таблиц DoclingDocument."""
    # Форматирует таблицы как текст с разделителями |
    return "\n".join(table_texts)
```

**Результат:** ИНН теперь извлекается из таблиц и включается в контент для LLM.

## 3. Работа INN Enrichment

### 3.1 Тестовый поиск ИНН

```bash
curl -X POST "http://localhost:8001/api/v1/qa/inn/search" \
  -H "Content-Type: application/json" \
  -d '{"org_name": "Ромашка"}'

# Результат:
{
  "org_name": "Ромашка",
  "inn": null,
  "found": false,
  "source": "web_search"
}
```

**Вывод:** Web search работает корректно, но требует точного названия организации.

### 3.2 Источники данных

ZaiSearchClient поддерживает поиск через:
- Rusprofile API (suggestions.rusprofile.ru)
- List-org.com (парсинг)

**Ограничения:**
- Требуется точное юридическое название
- "Заявка №XXXXX" не может быть обогащена
- Кэширование на уровне сессии

## 4. API Endpoints

### 4.1 Новые endpoints

```
POST /api/v1/qa/inn/search
    → Поиск ИНН по названию организации

POST /api/v1/qa/enrich-inn
    → Обогащение одного документа

POST /api/v1/qa/enrich-batch
    → Пакетное обогащение документов
```

### 4.2 Использование

```python
# Поиск ИНН
response = requests.post(
    "http://localhost:8001/api/v1/qa/inn/search",
    json={"org_name": "ООО Ромашка"}
)
# → {"inn": "1234567890", "found": true, "source": "rusprofile"}

# Обогащение документа
response = requests.post(
    "http://localhost:8001/api/v1/qa/enrich-inn",
    json={"unit_id": "UNIT_XXX", "force_search": false}
)
```

3. ✅ **DoclingDocument содержит полные данные** — нужно только корректно их извлекать

### Ключевое изменение

**MongoContextLoader** теперь извлекает данные из:
- Массива `texts` (как раньше)
- Массива `tables` (новое!)

Таблицы форматируются в текстовом виде с разделителями `|` и добавляются в контент с маркером `=== ТАБЛИЦЫ ИЗ ДОКУМЕНТА ===`.

```json
  "winner_inn": null
}
```

**После модификации:**
